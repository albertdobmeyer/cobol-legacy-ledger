GitHub Visibility Strategy
Problem: GitHub stars/forks are vanity metrics, but Anthropic engineers will look at your repos. Here's what actually matters to them:
What Anthropic engineers look for (not stars):

Commit history depth — Show iterative thinking, not one massive commit
Documentation clarity — Can a developer new to your code understand it?
Error handling philosophy — How do you handle edge cases?
Test coverage mindset — Do you think about correctness?
Architecture decisions — Can you articulate why you built it this way?
Issue discussions — Do you engage thoughtfully with problems?

To get authentic visibility (stars/engagement):
Strategy A: Content Marketing

Write 1 blog post per project: "Why I Built X, What I Learned"
Post to: dev.to, Hacker News, Reddit r/learnprogramming, Anthropic's Discord
Link back to GitHub
Share on LinkedIn with your Anthropic context
Example: "Building a Local-First AI Orchestrator: Lessons from Running Claude + Ollama Together"

Strategy B: Community Seeding

Join Anthropic's official Discord/community forums
Don't spam. Instead: answer 5 questions for every 1 project mention
Share projects when directly relevant to someone's problem
Build genuine relationships with other Claude enthusiasts
This = trust signal to Anthropic recruiters ("Oh, we know this person, they're thoughtful")

Strategy C: Demo Videos

Pick your 3 strongest projects
Record 2-3 min "I built this" demo videos
Host on YouTube shorts or Twitter/X
Show the problem being solved, not the code
Post on LinkedIn: "Just shipped [X]. Here's what it does..."
Videos get way more engagement than repos
